## How to run:

- Clone this repo locally and cd into it.

- Ensure ruby 4.0.5 is installed. I developed on Windows using:
https://rubyinstaller.org/downloads/

- With ruby installed run 'gem install bundler' in gitbash/PowerShell

- In the cloned directory run 'ruby bin/setup'. This should install requirements, setup the database, and spin up a local server. (After setting up once, you should be able to just run ruby bin/rails server to spin up again.)

- Open a browser and go to 'https://localhost:3000'

## Where I would take the project:

I would first want feedback from users on use case to avoid flooding the application with unnecessary features. Even the current feature that colors report tiles based on grade seems potentially unnecessary though I thought it improved the look of the page.

One obvious feature would be a tag system as well as sorting options further than just the search bar. Being able to sort players by position, team, grade, or custom tags would make the user experience much smoother especially as the list of reports grows.

I would also like to make better use of the dashboard screen space, currently it stores a card allowing users to quickly return to the most recently edited report, but there is definitely a lot more that could fit there before that panel feels cluttered. This is where user feedback would guide me, maybe it would be useful to have a calendar here that can store links to future player reports, that would probably come with another text input on report forms for reminders of what to look out for ahead of time. Or it might be useful to have a new 'group' data type here that stores custom groups of players that scouts know they will want to review together. This change would go well with the potential tag system, groups could be autogenerated based on shared tags.

Another feature could be user/login support so different users can have unique reports for the same players or leave comments/notes on other users reports.

## AI use

I went into this project familiar with general web-dev concepts, but completely new to Ruby. I developed this entirely in VS Code with the Codex extension installed which can answer questions and make suggestions inline in my project. Most of my interactions were prompting the chat with questions like "How would a search bar typically be done in Ruby" or "How does Ruby expect me to connect to my create/update/delete .erb files in index". Most of it's contributions can be seen in my use of Ruby frameworks I wouldn't have known about otherwise like turbo frames, flash notifications, and stimulus. The only time Codex independently wrote code was to fix an install error I was seeing with bin/setup where Windows was trying to run a ruby command directly.
